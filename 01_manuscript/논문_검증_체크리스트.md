### A-2. 탈락 위험 중간 (강력 권장)

- [ ] **데이터 confounding 인정 (§3.3 또는 §6.3)**

  - 방향: "C4는 NM 추가와 동시에 총 데이터 량이 137K→175K로 증가하여 두 요인을 완전히 분리하지 못하였다. 이는 본 연구의 한계이며, NM 동량(同量) 통제 실험은 향후 과제이다" 추가

- [ ] **Train/Val/Test 분할 비율 명시 (§3.1)**
  - 현재: 없음
  - 방향: AIHub 원본 분할 비율 또는 자체 분할 비율(예: 8:1:1) 명시

- [ ] **씬 샘플링 파라미터 근거 추가 (§3.2)**
  - 현재: 임계값 0.7, 2초 간격, 해밍 거리 8 — 선택 근거 없음
  - 방향: 예비 실험 결과 또는 선행 연구 기반 근거 한 문장씩 추가
  - 예: "히스토그램 임계값 0.7은 예비 실험에서 씬 경계 검출 정밀도를 최대화하는 값으로 설정하였다"

- [ ] **해상도 결과 재서술 (§5.4)**
  - 현재: "mAP는 0.911에서 약 0.915 수준으로 사실상 손실이 없었다(약 0.4%p 이내)"
  - 실제: E07(416px, ~0.915) > E01(640px, 0.911) → 오히려 소폭 향상
  - 수정: "640→416 해상도 축소에서 mAP 손실이 없었으며 오히려 약 +0.4%p의 소폭 향상이 관찰되었다. 이는 더 작은 입력 해상도가 이 데이터셋에서 일종의 정규화 효과를 가질 수 있음을 시사한다."
  - → 주장이 더 강해지고 실제 수치와도 정합

- [ ] **YOLO11l 발산 상세 서술 (§5.2)**
  - 현재: "반복 발산하여 수렴에 실패"
  - 방향: 재시도 횟수, 손실 추이(발산 에폭 등), 원인 분석(데이터 규모 대비 과용량) 추가

- [ ] **하이퍼파라미터 선택 근거 (§4.1)**
  - 현재: 수치만 나열
  - 방향: "Ultralytics YOLO11 기본 설정을 베이스라인으로 사용하였으며, lr0=0.001과 patience=30은 선행 연구[x]에서 유사 데이터셋에 효과적임이 확인된 값을 적용하였다"
